# greymoth-devkit

**Deterministic rules in, a shareable verdict card out.** The scoring engine and
the born-to-share card that had been hand-copied across ~28 greymoth tools —
extracted into one small toolkit.

No LLM. No network. No runtime dependencies. Same input, same card, every time.

```
rules-core  →  scorecard  →  sharecard
(evaluate)     (verdict)      (1200×630 SVG you can post)
            tokens — the identity underneath all three
```

[![license](https://img.shields.io/badge/license-MIT-9c3a2c)](./LICENSE) ·
zero-dependency · deterministic · Node ≥ 24

---

## Why this exists

Across a couple dozen small "checker" apps I kept rewriting the same three things:
a rule engine that turns a subject into a 0–100 score, a score→verdict map with a
rubber-stamp badge, and an OG card to make the result worth sharing. This is that
code, written once, as four packages that compose.

The constraints are the product: **no model calls, no network, no randomness.**
A score is reproducible and every point it deducts is attributable to a named rule.

## Quick start

```bash
npm i @greymoth/rules-core @greymoth/scorecard @greymoth/sharecard
```

```ts
import { evaluate, rule } from "@greymoth/rules-core";
import { scoreCardSvg } from "@greymoth/sharecard";

const rules = [
  rule({ id: "license", title: "LICENSE present", severity: "error", weight: 3, test: (r) => r.hasLicense }),
  rule({ id: "tests",   title: "Has a test suite", severity: "error", weight: 3, test: (r) => r.hasTests }),
  rule({ id: "ci",      title: "CI configured",   severity: "warn",  weight: 1, test: (r) => r.hasCi }),
];

const report = evaluate(rules, { hasLicense: true, hasTests: true, hasCi: false });
report.score; // 86

// score → verdict → a 1200×630 SVG you can drop into an <img> or an OG tag
const svg = scoreCardSvg({ score: report.score, title: "Repository health" });
```

That `svg` is a complete, self-contained image — rasterize it with any headless
browser or with `@vercel/og`'s Satori, or serve it as `image/svg+xml`.

## The packages

| Package | What it does | Deps |
|---|---|---|
| [`@greymoth/rules-core`](./packages/rules-core) | Define rules as data; `evaluate()` → findings + 0–100 score; `noisyOr()` for risk aggregation | none |
| [`@greymoth/scorecard`](./packages/scorecard) | `verdict()` score→{level,label,color} + the distressed verdict-stamp as SVG/HTML | tokens |
| [`@greymoth/sharecard`](./packages/sharecard) | born-to-share 1200×630 cards as zero-dep SVG strings, with a deterministic serial | tokens, scorecard |
| [`@greymoth/tokens`](./packages/tokens) | The greymoth identity (paper / ink / oxblood, IBM Plex Mono, grain, stamp) as CSS vars + typed JS | none |

### `@greymoth/rules-core`

Rules are plain objects; the evaluator is a pure function. Two aggregations:

```ts
import { evaluate, noisyOr, failures } from "@greymoth/rules-core";

// linear budget: 100 minus the weight of what failed (graded penalties supported)
const r = evaluate(rules, subject);
r.score;            // 0–100
failures(r);        // failing findings, worst-first — every point lost has a name

// noisy-OR: independent signals accumulate but saturate toward 1 (the llm-shield model)
const risk = noisyOr(injectionSignals, prompt); // { risk: 0.9, score: 90, matched: [...] }
```

### `@greymoth/scorecard`

```ts
import { verdict, verdictStampSvg, scorecardHtml } from "@greymoth/scorecard";

verdict(73);                 // { score: 73, level: "warn", label: "NEEDS WORK", color: "#c2790b" }
verdictStampSvg(verdict(30), x, y);   // <g> stamp fragment for an SVG
scorecardHtml({ score: 91, title: "Deliverability" }); // a self-contained HTML block
```

Bands and colors are overridable per call; the defaults are the median of what the
apps actually shipped.

### `@greymoth/sharecard`

```ts
import { shareCardSvg, scoreCardSvg, serialOf } from "@greymoth/sharecard";

shareCardSvg({ kicker: "greymoth · llm-shield", metric: "97", title: "Injection risk",
               verdict: { label: "BLOCK" } });
serialOf({ metric: "97" });  // "CARD-1A2B3C" — deterministic, no clock/random
```

SVG strings on purpose: they run on a server, an edge runtime, or in the browser,
embed nothing external, and are trivially testable (you can assert on the string).

### `@greymoth/tokens`

```ts
import { color, font, toCss } from "@greymoth/tokens";
color.oxblood; // "#9c3a2c"
// or pull the CSS:  import "@greymoth/tokens/tokens.css";  →  var(--gm-oxblood)
```

## Layout

```
greymoth-devkit/
├─ packages/
│  ├─ tokens/        @greymoth/tokens
│  ├─ rules-core/    @greymoth/rules-core
│  ├─ scorecard/     @greymoth/scorecard
│  └─ sharecard/     @greymoth/sharecard
├─ examples/         the end-to-end demo + generated cards
└─ index.html        the story / showcase page
```

## Develop

```bash
npm install      # link workspaces
npm run build    # tsc all packages (+ generate tokens.css)
npm test         # build, then node:test across every package
npm run demo     # regenerate the example cards
```

TypeScript, ESM, Node's built-in test runner. No bundler, no jest, no eslint config —
`tsc --strict` is the gate.
