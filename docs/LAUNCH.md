# LAUNCH — distribution checklist & copy

The point of this repo (per the 2026-06-22 portfolio audit) is **distribution**, not
another build. The code is done; what converts it into reputation is shipping it.
Everything here is ready except the steps that need a human login.

## Pre-publish (do once)

- [ ] `npm whoami` — log in; create the `@greymoth` npm org/scope if needed.
- [ ] Copy `LICENSE` into each `packages/*/` (npm includes the field already; a file is tidier).
- [ ] Decide public vs. scoped-public: `npm publish --access public` for each package.
- [ ] Push to GitHub: `gh repo create greymoth-jp/greymoth-devkit --public --source=. --push`.
- [ ] Enable GitHub Pages on `main` (root) → the story page at `index.html` goes live.

## Publish order (respects dependencies)

```bash
npm run build && npm test          # must be green
npm publish -w @greymoth/tokens --access public
npm publish -w @greymoth/rules-core --access public
npm publish -w @greymoth/scorecard --access public   # depends on tokens
npm publish -w @greymoth/sharecard --access public   # depends on tokens, scorecard
```

### Show HN body
> I kept rewriting the same three things across ~28 small "checker" apps: a rule
> engine that turns a subject into a 0–100 score, a score→verdict map with a stamp,
> and an OG card to make the result worth sharing. So I extracted them into four
> zero-dependency TypeScript packages that compose: rules-core → scorecard →
> sharecard, on a shared tokens identity.
>
> The constraint is the product — no model calls, no network, no randomness. Scores
> are reproducible and every point deducted is attributable to a named rule. The
> share card is a plain SVG string that runs on a server, an edge runtime, or in the
> browser. The three cards on the page were generated by the code, not drawn.
>
> Repo + live page: <link>. Feedback on the API welcome.
